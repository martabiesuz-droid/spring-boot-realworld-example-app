# Definition of Ready (DoR)

An issue is considered **Ready** to start work when it meets all of the following:

- [ ] Has a clear, descriptive title
- [ ] Has a description explaining the motivation (why this is needed)
- [ ] Has explicit, testable acceptance criteria (a checklist of what "done" looks like)
- [ ] Scope is clear: what is included and what is explicitly out of scope
- [ ] Any dependency on another issue or repository is identified and linked
- [ ] Any known technical risk or open question is flagged in the issue body or a comment, before work starts
- [ ] The branch this work will live on is identified (new branch or existing feature branch)

If any of the above is missing, the issue should be refined before implementation starts — not decided mid-implementation.
